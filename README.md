# Second Brain

> Локальна система управління знаннями на базі Markdown + Claude Code.
> Побудована для людей з активним стилем життя: CTO, дослідники, спортсмени, мандрівники.

---

## Що це таке

**Second Brain** — це персональна база знань яка живе локально у вас на комп'ютері як звичайні Markdown файли. Ніяких підписок, ніяких хмар, ніяких lock-in.

Система побудована на двох стовпах:
- **[Obsidian](https://obsidian.md)** — для читання, редагування та навігації нотатками
- **[Claude Code](https://claude.ai/code)** — як AI-асистент, тренер, аналітик і радник прямо всередині вашої бази знань

Архітектура базується на методі **PARA** (Projects → Areas → Resources → Archive) плюс щоденні нотатки як центр управління.

## Швидкий старт

### 1. Клонуй репозиторій

```bash
git clone https://github.com/your-username/second_brain.git
cd second_brain
```

### 2. Встанови Claude Code

```bash
npm install -g @anthropic-ai/claude-code
```

> Потрібен акаунт на [claude.ai](https://claude.ai). Claude Code використовує файл `.claude/CLAUDE.md` як інструкції — система вже налаштована.

### 3. Відкрий в Obsidian

1. Завантаж [Obsidian](https://obsidian.md) якщо ще немає
2. `Open folder as vault` → вибери папку `second_brain/`
3. Встанови три плагіни (Settings → Community Plugins):
   - **Templater** → вкажи папку шаблонів: `_Templates`
   - **Dataview** → для SQL-подібних запитів до нотаток
   - **Calendar** → для навігації по щоденних нотатках

### 4. Персоналізуй

Відкрий `.claude/CLAUDE.md` і замін контекст під себе:
```
- Власник — [твоя роль]
- Мова нотаток: [мова]
- Ключові сфери: [твої сфери]
```

```
│   │   └── Strategy/         ← Tech radar, бачення, плани
│   ├── Research/
│   │   ├── Papers/
│   │   ├── Experiments/
│   │   └── Notes/
│   ├── Health/

│
├── 04_Knowledge/              ← База знань (reference, не проекти)
│   ├── Books/                 ← Нотатки з книг
│   ├── Videos/                ← Нотатки з відео, подкастів, курсів
│   ├── Ideas/                 ← Сирі ідеї для розвитку
│   └── Concepts/             ← Ключові концепції та моделі
│
├── 05_Archive/                ← Завершені проекти, старі нотатки
```

**Правило PARA:** якщо є чіткий кінець → `02_Projects`. Якщо немає → `03_Areas`. Якщо просто знання → `04_Knowledge`. Не впевнений → `00_Inbox`.

## Робочий процес

### Щодня — 10 хв

**Ранок (5 хв)**
1. Нова нотатка з шаблону `Daily.md` → назва `YYYY-MM-DD.md`
2. Встанови **один** головний фокус дня
3. Перелік задач по сферах

**Ввечері (5 хв)**
1. Позначити виконані задачі
2. Заповнити тренування та харчування
3. Рефлексія: енергія / настрій / сон (1-10)

### Щотижня — 20 хв

1. **Оброби Inbox** — кожен файл або видали, або перемісти у правильне місце
2. **Оновити Projects** — статус, прогрес, блокери
3. **Action Items** — знайди невиконані задачі з нотаток зустрічей

```bash
# Claude зробить тижневий огляд за тебе
claude "тижневий огляд: незакриті задачі, action items із зустрічей, прогрес тренувань"
```

### Щомісяця — 30 хв

1. Аналіз здоров'я: тренування, харчування, БАДи
2. Архівуй завершені проекти в `05_Archive/`
3. Переглянь `04_Knowledge/Ideas/` — що готове до розвитку?

## Claude Code — приклади використання

Запускай прямо з папки `second_brain/`:

```bash
cd second_brain
claude "[запит]"
```

### Система / Inbox

```bash
# Обробити inbox
claude "переглянь 00_Inbox і запропонуй куди перемістити кожен файл"

# Тижневий огляд
claude "зроби тижневий огляд: незакриті задачі, зустрічі та рішення за цей тиждень"

# Знайти пов'язані нотатки
claude "знайди всі нотатки пов'язані з темою 'distributed systems'"

# Місячний огляд здоров'я
claude "проаналізуй мій прогрес у тренуваннях та харчуванні за квітень"
```

### Створення нотаток

```bash
# Нотатка з книги
claude "створи нотатку книги 'Designing Data-Intensive Applications' Kleppmann — головні ідеї: [ідеї]"

# Нотатка тренування
claude "запиши тренування: сьогодні біг 8км за 42хв, пульс середній 155"

# Захопити ідею
claude "захопи ідею: [ідея] — додай в 04_Knowledge/Ideas/"
```

## Скіли — AI радники

Система має 7 вбудованих AI-персон. Активуй потрібну роль прямо в запиті.

### Здоров'я

**Тренер (бодібілдінг)**
```bash
claude "як тренер, склади мені програму на гіпертрофію на 4 дні на тиждень"
claude "як тренер, поглянь на мої тренування за місяць і скажи що змінити"
claude "як тренер, я їду у відрядження на тиждень — програма без залу"
```

**Нутриціолог**
```bash
claude "як нутриціолог, проаналізуй моє харчування за цей тиждень"
claude "як нутриціолог, я на дефіциті 500ккал але стомлений — що не так?"
claude "як нутриціолог, що їсти під час 3-денної конференції без своєї кухні"
```

**Спортивний психолог**
```bash
claude "як спортивний психолог, немає мотивації йти в зал вже 2 тижні"
claude "як спортивний психолог, готуюся до першого тріатлону — mental prep"
claude "як спортивний психолог, відчуваю вигорання від роботи і спорту одночасно"
```

### Подорожі

**Тревел менеджер**
```bash
claude "як тревел менеджер, плануй поїздку в Берлін на 4 дні: конференція + тренування"
claude "як тревел менеджер, jet lag протокол для перельоту Київ → Сан-Франциско"
claude "як тревел менеджер, що взяти в місячну поїздку з тренуваннями і роботою"
```

### Наука

**Борд науковців (CS / IoT / High Load)**

Учасники: Jeff Dean, Martin Kleppmann, Liz Fong-Jones, Peter Norvig (CS) · Sanjay Sarma, Jan Rabaey (IoT) · Werner Vogels, Bryan Cantrill, Charity Majors (High Load)

```bash
claude "борд науковців: які ризики безпеки в IoT системі де [опис]"
claude "борд науковців: eventual consistency vs strong consistency для нашого use case: [опис]"
```

### Менеджмент

**Борд менеджерів технологічних компаній**

Учасники: Cal Henderson (Slack CTO) · Linus Torvalds (Linux) · Patrick Collison (Stripe CEO) · Mitchell Hashimoto (HashiCorp) · Kelsey Hightower (Google) · Adam D'Angelo (Quora, ex-Facebook CTO)

```bash

```

**1-on-1 аналізатор та менеджерські скіли**
```bash
# Підготовка до зустрічі
claude "підготуй мене до 1-on-1 з senior engineer який здається демотивованим останні 2 тижні"

# Аналіз після зустрічі
claude "проаналізуй цей 1-on-1: [вставити нотатки] — що важливе, які сигнали?"

# Важкий feedback
claude "як дати Radical Candor feedback тімліду що зривав дедлайни 3 рази підряд"

# Skip-level
claude "провів skip-level з командою, ось нотатки: [нотатки] — які системні проблеми бачиш?"
```

### Комбіновані запити

```bash

```

## Obsidian — корисні Dataview запити

Встанови плагін Dataview і використовуй ці запити в будь-якій нотатці:

```dataview
// Активні проекти
TABLE date_created, status, priority FROM "02_Projects"
WHERE status = "active"
SORT priority ASC
```

```dataview
// Останні тренування
TABLE date, sport, duration FROM "03_Areas/Health/Training"
SORT date DESC
LIMIT 10
```

```dataview
// Незакриті рішення
TABLE date, area, impact FROM "03_Areas/CTO/Decisions"
WHERE status = "open"
```

```dataview
// Ідеї готові до розвитку
TABLE date, area FROM "04_Knowledge/Ideas"
WHERE status = "ready"
```

```dataview
// Книги в процесі читання
TABLE author, rating FROM "04_Knowledge/Books"
WHERE status = "reading"
```

## Принципи системи

1. **Inbox First** — не думай де покласти, просто захопи. Сортуй раз на тиждень.
2. **Projects vs Areas** — є дедлайн і результат → Project. Немає кінця → Area.
3. **Daily — центр управління** — все важливе проходить через щоденну нотатку.
4. **Одна нотатка = одна одиниця** — одна ідея, одна зустріч, одне тренування. Не мегадокументи.
5. **Frontmatter завжди** — `date`, `type`, `status`, `tags` — це дає пошук і Dataview.
6. **Claude читає контекст** — перед порадою Claude читає твої реальні нотатки, а не відповідає в загальному.

## Персоналізація під себе

Система адаптована під CTO + науковець + спортсмен + мандрівник. Щоб адаптувати під себе:

1. **Змін `.claude/CLAUDE.md`** — опиши свою роль, сфери, мову
2. **Видали непотрібні Area папки** — залиш тільки свої сфери
3. **Змін або видали скіли** — замін персон борду на релевантні для твоєї галузі
4. **Адаптуй шаблони** — кожен шаблон в `_Templates/` можна редагувати під свій стиль
